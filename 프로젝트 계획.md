

##가능한 기초 분석 및 지표 선정

1. 유저 통계 분석(user info based segmentation) - user dashboard

유저의 성별 분포: 'gender' 컬럼을 기반으로 성별에 따른 유저 수를 확인
국가별 유저 분포: 'country' 컬럼을 통해 국가별로 유저 수를 확인
가입 일자 분포: 'first_login_date' 컬럼을 통해 유입이 가장 많았던 혹은 적었던 시기를 확인

2. 결제 관련 분석(pay amount based segmentation): - sales dashboard

결제 플랫폼별 결제량 비교: 'pay_store' 컬럼을 기반으로 각 결제 플랫폼별 결제량(or 결제건수)을 비교
결제 유저 유형 분석: 'pay_user_type' 컬럼을 통해 각 유형의 유저의 평균 결제 금액 비교
평균 결제 금액 분석: 'pay_amount' 컬럼을 통해 전체 평균 결제 금액을 계산하여 게임 내 매출을 이해합니다. 위의 분석과 같이 표시

3. 유저 행동 분석(user action based segmentation): - user dashboard

레벨별 플레이어 분포: 'level' 컬럼을 기반으로 각 레벨의 플레이어 수 분포를 확인(레벨 구간 별 세그멘테이션도 가능)
행동 유형에 따른 분석: 'action_user_type' 컬럼을 통해 유저의 행동 분포를 대략적으로 파악, 인기 있는 컨텐츠 파악
플레이타임 분석: 'playtime' 컬럼을 통해 평균 플레이타임을 계산하고, 전체적인 플레이 타임 분포도 파악


5. Battle Pass 분석: - sales dashboard + user dashboard

Battle Pass 구매율 분석 : 'battle_pass_level' 컬럼을 통해 전체 유저 중 패스 구매 유저 비율을 확인
Battle Pass 레벨별 참여율 분석: 'battle_pass_level' 컬럼을 통해 Battle Pass에 참여한 유저의 전체적인 진행 분포 확인
Battle Pass와 결제 간의 상관관계 분석: Battle Pass 참여자들과 그렇지 않은 유저들 간의 결제 패턴을 비교하여 
                                     Battle Pass가 결제에 미치는 영향을 확인합니다.(고,중,저과금 유저 세분화도 가능)

6. 게임 구조 안정성 분석(성장 및 성과, 레벨링 구조 설계 평가): - 인사이트로 이어질 가능성

플레이타임과 레벨 간의 상관관계 분석: 게임을 많이 플레이한 유저가 어느 정도 레벨에 도달하는지 확인하여 
                                   설계된 게임의 레벨 디자인이 올바르게 작동하고 있는지 평가
플레이타임과 결제 간의 상관관계 분석: 게임을 오래 플레이한 유저가 더 많이 결제하는지를 확인하여 게임 플레이의 
                                   경제적인 영향을 파악.
